Skills are capabilities you toggle on per agent — closer to apps than to plugins. Each skill bundles instructions, tools, and trigger conditions: turn on the browser skill and the agent can drive a real browser; turn it off and those tools vanish, so the agent can't call them by accident. Froots ships with a large library — browser automation, transcription, image and video generation, email, calendars, messaging channels like Telegram and Slack, and dozens more — and agents stay honest about what's currently available to them.

This page is an outline — the full guide is being written. It will cover:

- Browsing the skill library and toggling skills per agent
- Skills that need API keys or accounts, and where those go
- Installing community skills — and vetting them before you do
- Writing your own skill

The [Skills](/skills) page has the current overview, and [Designing the Skills library](/blog/skills-library-design) on the blog explains why skills work this way.
